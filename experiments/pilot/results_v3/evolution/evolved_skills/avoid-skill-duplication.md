## Prevent Duplicate Skill Creation

1. When asked to generate new skills, you will be given a list of existing skill names.
2. Carefully review this list before starting your work.
3. Your primary goal is to create *novel* skills that address a gap.
4. Do not generate a skill that has the same name or covers the exact same concept as an existing one. A slight rewording is still a duplicate.

**Anti-pattern:** Generating a skill like `propose-document-outline` when `propose-outline-for-structured-output` already exists.
