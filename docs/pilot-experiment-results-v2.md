### 3.1 Week 1: 학습자 분석

#### Baseline

**T (용어 정확성): 4/5**  
"출발점 행동(Entry Behaviors)", "사전 지식(Prior Knowledge)", "자기 효능감(Self-efficacy)", "KWL 차트" 등 Dick & Carey 모형의 핵심 용어를 정확히 사용. 다만 "학습 스타일"을 VARK 등 특정 모형에 연결하지 않고 일반적으로 서술.

**C (내용 완전성): 4/5**  
요구된 3개 영역(출발점 행동, 사전 지식, 학습자 특성)을 모두 포함하고, 각각에 데이터 수집 방법도 구체적으로 제시. 단, "오개념(misconception)" 분석이 사전 지식 영역에만 포함되고 출발점 행동에서는 다루지 않음. 수준별 분류(상/중/하)가 없이 "중상" 수준이라고 뭉뚱그려 기술.

**M (모형 정합성): 3/5**  
Dick & Carey 모형의 학습자 분석 단계를 기본적으로 따르고 있으나, 모형에서 중시하는 "목표 행동(target behaviors)"과의 연결이 약함. 학습자 분석이 후속 단계(격차 분석, 학습목표)에 어떻게 활용될지에 대한 명시적 언급이 부족.

**P (실용성): 3/5**  
진단평가, KWL 차트, 설문조사 등 구체적 도구를 제시했으나, 문항 예시나 실제 척도 내용이 없어 "어떤 진단평가를 하라"까지는 알 수 있지만 바로 실행하기엔 부족. 시간 계획(20분 진단평가 등)은 현실적.

**L (맥락 연결성): 3/5**  
주어진 맥락(고2, 미적분, 30명, 주 4시간)을 보고서에 반영했으나, 맥락의 특수성(예: 한국 교육과정에서 미적분이 어느 시점에 배치되는지, 수능과의 관계)을 깊이 있게 다루지 않음.

## 5. 논의

### 5.1 Seed Skill의 효과

Skill 주입은 **Week 1(학습자 분석)에서 가장 효과적**이었으며, 특히 **내용 완전성과 실용성** 측면에서 개선을 가져왔다. 이는 학습자 분석이 상대적으로 구조화된 태스크이고, seed skill이 제공한 프레임워크(3영역 구분, 데이터 수집 방법 명시)가 모델의 출력 구조를 직접적으로 안내했기 때문으로 해석된다.

반면 Week 2~3에서는 skill 효과가 거의 나타나지 않았다. 이는 두 가지 원인이 있다:
1. **체이닝 효과**: Week 2~3는 이전 주차 output을 input으로 받으므로, 모델의 성능이 seed skill보다 이전 산출물의 품질에 더 크게 의존
2. **천장 효과**: Baseline Week 3도 이미 24/25로 높은 수준이라 개선 여지가 적음

### 5.2 자기 평가 편향 문제

Gemini 2.5 Pro가 자기 자신의 출력을 평가했을 때 **모든 산출물에 25/25 만점**을 부여한 것은 심각한 자기 평가 편향(self-evaluation bias)이다. 이는 ISD-Bench의 평가 설계에 중요한 시사점을 준다:

- **초안의 LLM-as-judge 검증(Section 5.3) 설계가 매우 중요** — 같은 모델 또는 같은 계열 모델이 평가하면 편향이 발생
- **인간 전문가 평가가 불가피** — 최소한 평가 기준 calibration과 편향 검증을 위해
- **교차 평가(cross-evaluation)** 설계 필요: 모델 A의 출력을 모델 B가 평가

### 5.3 ISD-Bench 연구에 대한 시사점

1. **Skill 효과가 가장 큰 구간**: 초반 구조화된 태스크(Week 1~2)에서 skill이 효과적. MetaClaw 논문의 "중간 난이도에서 adaptation 효과 극대화" 발견과 일부 일치.

2. **체이닝의 중요성**: Week 2~3에서는 이전 산출물의 품질이 현재 산출물을 좌우. ISD-Bench에서 15주 누적 시 초기 오류가 복리로 쌓이는 문제(초안 Section 11에서 언급)가 실제로 발생할 수 있음.

3. **평가 모델 분리 필수**: 생성 모델과 평가 모델은 반드시 다른 모델/계열을 사용해야 함.

4. **Baseline이 이미 높은 수준**: Gemini 2.5 Pro의 ISD 기본 역량이 높아서, skill의 차별적 효과를 관찰하려면 더 어려운 태스크(Week 5~7)이나 더 엄격한 평가 기준이 필요.

## 6. 연구 설계 수준의 인사이트

### 6.1 ISD에서 Skill 효과의 본질은 CLI와 다르다

파일럿에서 skill 주입 효과는 Week 1에서만 +3점, Week 2~3은 0점이었다. 이는 MetaClaw 논문의 "중간 난이도에서 효과 극대화" 발견과 다른 패턴이다. 원인은 ISD 태스크의 성격 차이에 있다:

| | CLI 태스크 (MetaClaw-Bench) | ISD 태스크 (ISD-Bench) |
|---|---|---|
| 핵심 요구 | 규칙 준수 (날짜 포맷, .bak 생성 등) | 전문 지식의 깊이 (교수설계 원칙 적용) |
| Skill의 역할 | 정답 규칙을 직접 알려줌 | 구조를 안내하지만 내용 품질은 사전 지식에 의존 |
| Skill 효과 | 정답률을 직접 올림 | 구조화를 돕지만 전문성의 깊이는 올리지 못함 |

**시사점**: 초안의 C4 조건에서 MetaClaw Skills의 효과가 CLI 도메인만큼 크지 않을 가능성이 있다. 이것이 논문의 차별적 기여가 될 수 있다 — **"skill-driven adaptation은 규칙 기반 태스크에서는 효과적이지만, 전문 지식 기반 태스크에서는 제한적"**이라는 발견.

### 6.2 체이닝(chaining)이 adaptation보다 강력한 변수

Week 2~3에서 baseline과 skills의 점수가 동일(22, 24)한 것은, 두 조건 모두 이전 주차 output을 input으로 받았기 때문이다. 이는 초안의 RQ3("주차 누적에 따라 격차가 감소하는가?")에 대해 중요한 복잡성을 드러낸다:

```
초안의 가정:
  Skill 누적 → 에이전트 성능 향상 → 인간-AI 격차 감소

실제 가능성:
  이전 산출물 품질 → 현재 산출물 품질에 더 큰 영향
  → 초기(Week 1~3) 품질이 높으면 후기도 자동으로 높아짐
  → Skill 효과와 체이닝 효과를 분리하기 어려움
```

**시사점**: RQ3를 제대로 답하려면, 체이닝 효과를 통제하는 조건이 필요하다:
- 조건 A: 매주 이전 산출물을 그대로 전달 (현재 설계)
- 조건 B: 매주 "표준 산출물"을 전달 (체이닝 통제)
- A와 B의 차이가 체이닝 효과, B에서의 주차별 변화가 순수 skill 효과

### 6.3 자기 평가 편향은 "연구 질문"이 될 수 있다

Gemini가 자기 출력에 전원 만점(25/25)을 준 것은 기술적 문제가 아니라, 초안의 LLM-as-judge 검증(Section 5.3)에 대한 중요한 예비 데이터다.

| 평가 설계 | 결과 | 변별력 |
|----------|------|:---:|
| Gemini → Gemini 출력 (자기 평가) | 전원 25/25 만점 | 없음 |
| Claude → Gemini 출력 (교차 평가) | 17~24/25 범위 | 있음 |

초안에서 추가한 "LLM judge는 ISD 산출물 평가에서 인간 전문가 수준의 신뢰도를 달성하는가?"에 대해 이미 하나의 답을 얻었다: **동일 모델 자기 평가는 변별력이 없다.** 이 "자기 평가 편향 vs 교차 평가" 분석 자체가 교육 도메인에서 LLM-as-judge의 신뢰도를 체계적으로 분석한 연구로서 별도 기여 가능성이 있다.

### 6.4 MetaClaw의 활용: 설정 문제 → 해결 → ISD Skill 자동 합성 성공

#### 오진 경위와 수정

초기 실험(v1)에서 skill evolution이 트리거되지 않아 "skills_only 모드에서는 PRM이 비활성이라 구조적으로 불가"로 결론을 내렸으나, 이는 **오진**이었다.

코드 분석(`api_server.py:2080`) 결과, skills_only 모드에서도 `reward=0.0`으로 모든 턴을 evolver에게 보내어 **패턴 분석 기반 skill 합성**이 가능하도록 설계되어 있었다. 실패 원인은 3가지 설정 오류:

| 원인 | 상세 | 해결 |
|------|------|------|
| Config 필드명 불일치 | `skills.auto_evolve` 설정 → 실제 필드는 `enable_skill_evolution` | 정확한 필드명으로 재설정 |
| Evolver API 미설정 | `evolver_api_base`, `evolver_api_key` 미지정 → 초기화 실패 | 프록시(localhost:30000)를 evolver endpoint로 설정 |
| turn_type 기본값 함정 | session_id를 명시하면 turn_type 기본값이 `"side"` → 버퍼 미적재 | `turn_type=main` 명시 |

#### 실험 3 v3 결과: 8개 ISD Skill 자동 합성

올바른 설정 후 5개 ISD 태스크(ARCS, Gagné, 형성평가, Merrill, 차별화 교수)를 전송한 결과, **4회 evolution이 발생하여 총 8개 skill이 자동 합성**되었다.

| Skill | 카테고리 | ISD 관련성 |
|-------|---------|----------|
| `apply-instructional-design-frameworks` | general | ISD 모형(Merrill, Tomlinson)의 구성요소를 응답 구조로 사용 |
| `integrate-dick-and-carey-model` | general | ARCS, Gagné 등을 Dick & Carey의 상위 프레임워크 안에 배치 |
| `uphold-instructional-design-persona` | agentic | 요청된 하위 모형이 있어도 핵심 Dick & Carey 페르소나 유지 |
| `produce-formal-pedagogical-documents` | communication | 학술적 문서 형식(계층 구조, 표, 루브릭) 준수 |
| `propose-outline-for-structured-output` | agentic | 복잡한 ISD 문서 작성 전 개요를 먼저 제안 |
| `match-request-language` | communication | 응답 언어를 프롬프트 언어와 일치 (이전 중국어 전환 문제 해결) |
| `avoid-skill-duplication` | common_mistakes | 기존 skill과 중복되는 새 skill 합성 방지 (메타 skill) |
| `focus-on-failure-pattern` | agentic | 실패 분석 시 주제가 아닌 행동 패턴에 집중 (메타 skill) |

#### Evolution 타임라인

```
태스크 1~3 완료 (ARCS, Gagné, 형성평가)
  ↓ 3턴 누적 → 1차 evolution 트리거 (비동기)
태스크 4 완료 (Merrill)
태스크 5 완료 + session_done=True (차별화 교수)
  ↓ 세션 종료 → 2차~4차 evolution 연쇄 트리거
─────────────────────────────────────────
결과: 4회 evolution × 2개/회 = 8개 skill 합성
```

**주목할 점**: 스크립트의 `check_evolved_skills()` 시점에서는 "합성된 skill 없음"으로 표시되었으나, evolution이 **비동기적**으로 실행되어 스크립트 종료 후 skill 파일이 생성됨. 실제 검증은 `~/.metaclaw/skills/` 디렉토리와 `evolution_history.jsonl`에서 확인.

#### 합성된 Skill의 분석

**ISD 도메인 특화 (6/8 = 75%)**:
합성된 8개 skill 중 6개가 ISD 원칙에 직접 관련된 내용이다. 특히 `integrate-dick-and-carey-model`과 `uphold-instructional-design-persona`는 시스템 프롬프트에 "Dick & Carey 모형 기반"이라고만 명시했는데, evolver가 이를 발전시켜 **하위 모형(ARCS, Gagné)과의 통합 전략**이라는 더 구체적인 행동 규칙으로 합성한 것이다.

**범용 메타 skill (2/8 = 25%)**:
`avoid-skill-duplication`과 `focus-on-failure-pattern`은 ISD 특화가 아닌 evolver 자체의 동작을 개선하는 메타 skill이다. MetaClaw의 evolver가 자기 참조적(self-referential) skill을 합성하는 능력을 보여준다.

**언어 문제 자동 해결**:
`match-request-language` skill은 v1 실험에서 발견된 "형성평가 태스크 중국어 응답" 문제를 evolver가 자동으로 인식하고 해결 규칙을 합성한 것이다. 이는 skill evolution이 **관찰된 문제를 자동 교정**할 수 있음을 보여주는 중요한 사례.

#### 시사점 수정

~~"MetaClaw를 있는 그대로 활용 불가"~~ → **올바르게 설정하면 skills_only 모드에서도 ISD 도메인 skill evolution이 정상 동작한다.** 다만 다음 주의가 필요:
1. `enable_skill_evolution` (config 필드명 정확히), `evolver_api_base/key/model` 설정 필수
2. `session_id` 사용 시 반드시 `turn_type=main` 명시
3. Evolution은 비동기 — 결과 확인은 skill 디렉토리/history 파일에서

### 6.5 평가 기준 "실용성(P)"이 가장 변별력이 높다

5개 기준 중 실용적 적용 가능성(P)이 baseline vs skills에서 가장 큰 차이(+0.7)를 보였다:

```
변별력 순서: P (실용성, +0.7) > C (완전성, +0.3) > M = T = L (0.0)
```

**시사점**: 본 실험에서 "실용성"을 세분화하면 skill 효과를 더 민감하게 포착할 수 있다:
- P1: 구체적 도구/방법 명시 여부
- P2: 시간/자원 제약 반영 여부
- P3: 실제 문항/체크리스트 예시 포함 여부

### 6.6 논문 포지셔닝 강화 포인트

| 초안의 기존 기여 | 파일럿에서 추가된 근거 |
|----------------|---------------------|
| ISD 도메인 최초 벤치마크 | Baseline 품질이 양호(21/25)하여, 벤치마크의 가치가 "가능/불가능"이 아니라 "어떤 조건에서 얼마나 좋아지는가"에 있음 확인 |
| MetaClaw의 교육 도메인 확장 | 프록시 직접 활용 한계 확인 → skill 로직만 추출하는 "도메인 적응" 방법론 자체가 기여 |
| 인간-AI 종단 비교 | 체이닝 효과 발견 → "체이닝 vs skill 효과 분리" 분석이 추가 가치 |
| LLM-as-judge 검증 | 자기 평가 편향 실증 → 교육 도메인에서의 LLM 평가 신뢰도 자체가 별도 기여 |
