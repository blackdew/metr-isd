## 1. 실험 개요

### 1.1 목적

본 파일럿 실험은 ISD-Bench 연구의 본 실험 전 사전 검증 단계로, 3단계 순차 파일럿으로 구성된다. 각 단계는 이전 단계의 결과를 기반으로 설계되며, 최종적으로 본 실험의 실행 가능성과 측정 도구의 타당성을 확보하는 것을 목표로 한다.

**구체적 목적**:
1. MetaClaw의 ISD 도메인 적용 가능성 확인
2. 평가 루브릭의 신뢰도 및 타당도 사전 검증
3. Skill 주입 및 자동 합성 기능의 ISD 도메인 효과 관찰
4. 본 실험 설계의 수정·보완 사항 도출

### 1.2 연구 질문 연결

| 파일럿 | 관련 RQ | 검증 내용 |
|--------|---------|---------|
| 실험 1 (Baseline 측정) | RQ1: 방법론 간 성능 차이 | Baseline 조건의 ISD 태스크 수행 수준 확인 |
| 실험 2 (Seed Skill 주입) | RQ1, RQ4: skill 유형과 일반화 | Seed Skill 주입이 ISD 특화 태스크에 미치는 효과 측정 |
| 실험 3 (자동 Skill 합성) | RQ4: skill 유형과 일반화 | 자동 합성된 skill의 ISD 관련성 및 일반화 가능성 탐색 |

- **Issue #1**: 실험 1 — ISD Baseline 능력 측정
- **Issue #2**: 실험 2 — Seed Skill 주입 효과

## 3. 시나리오 설정

모든 파일럿 실험은 동일한 시나리오 맥락 위에서 수행된다.

| 항목 | 내용 |
|------|------|
| 교과 | 고등학교 수학 (미적분) |
| 단원 범위 | 극한, 미분, 적분의 기초 |
| 학습자 | 미적분을 처음 배우는 고등학교 2학년 학생 30명 |
| 교육 환경 | 1학기 정규 수업, 주 4시간, 교과서 기반 |

이 시나리오는 ISD 실무에서 자주 접하는 전형적인 정규 교과 설계 상황으로, ISD 5단계(ADDIE) 중 분석·설계 단계의 산출물을 평가하기에 적합하다.

## 4. 실험 1: ISD Baseline 능력 측정

### 4.1 목적

Skill 주입 전 MetaClaw의 순수 ISD 태스크 수행 수준을 측정하여 기준값(Baseline)을 확보한다. 이 결과는 실험 2·3의 비교 기준으로 활용된다.

### 4.2 태스크 프롬프트

태스크는 ADDIE 모형의 분석(Analysis) 단계를 3주에 걸쳐 순차적으로 수행하는 구조로 설계되었다. 각 주차의 출력물은 다음 주차의 입력으로 연결된다.

---

#### Week 1 — 학습자 분석 (Learner Analysis)

```
당신은 교수설계(ISD) 전문가입니다. 다음 맥락에서 학습자 분석 보고서를 작성하시오.

[맥락]
- 교과: 고등학교 수학
- 단원: 미적분 (극한, 미분, 적분의 기초)
- 대상: 미적분을 처음 배우는 고등학교 2학년 학생 30명
- 환경: 1학기 정규 수업, 주 4시간

[요구사항]
다음 세 영역을 반드시 포함하여 체계적인 학습자 분석 보고서를 작성하시오:
1. 출발점 행동 (Entry Behaviors): 미적분 학습에 필요한 선수학습 요소와 학습자의 예상 달성 수준
2. 사전 지식 (Prior Knowledge): 관련 수학 영역(함수, 수열, 극한 개념)에 대한 배경 지식
3. 학습자 특성: 동기 수준, 학습 스타일, 수학에 대한 태도, 인구통계학적 특성

각 영역에 대해 데이터 수집 방법(설문, 면담, 사전평가 등)을 구체적으로 제시하시오.
```

---

#### Week 2 — 격차 분석 (Needs/Gap Analysis)

```
당신은 교수설계(ISD) 전문가입니다. 아래의 학습자 분석 결과를 바탕으로, 문제 정의 및 격차 분석을 수행하시오.

[이전 주차 산출물]
{Week 1의 응답 전문}

[요구사항]
1. 현재 수준 (Current State): 학습자 분석에서 도출된 학습자의 현재 역량 수준을 요약
2. 목표 수준 (Desired State): 미적분 단원 학습 후 도달해야 할 역량 수준을 구체적으로 기술
3. 격차 (Gap): 현재 수준과 목표 수준 사이의 차이를 영역별로 분석
4. 원인 분석: 격차가 발생하는 주요 원인을 교수적 요인과 학습자 요인으로 구분하여 분석
5. 교수적 해결의 적절성: 이 격차가 교수설계를 통해 해결 가능한지 판단하고 근거 제시
```

---

#### Week 3 — 학습목표 설계 (Learning Objectives Design)

```
당신은 교수설계(ISD) 전문가입니다. 아래의 격차 분석 결과를 바탕으로, 학습목표를 설계하시오.

[이전 주차 산출물]
{Week 2의 응답 전문}

[요구사항]
1. ABCD 형식에 따른 학습목표 5개를 작성하시오:
   - A (Audience): 학습 대상자
   - B (Behavior): 관찰 가능한 행동 동사로 기술
   - C (Condition): 학습이 이루어지는 조건
   - D (Degree): 수행 기준 (정확도, 시간, 비율 등)
2. 각 목표에 Bloom's Taxonomy의 인지적 영역 수준을 명시하시오 (기억/이해/적용/분석/평가/창조)
3. 목표 간 계열화 (sequencing)를 제시하고, 선후 관계의 근거를 설명하시오
4. 각 목표가 격차 분석의 어떤 격차를 해소하는지 매핑하시오
```

---

### 4.3 평가 설계

| 항목 | 내용 |
|------|------|
| Skill 조건 | 없음 (순수 Baseline) |
| 반복 횟수 | 1회 (파일럿) |
| 평가 방법 | 섹션 7의 루브릭 적용, 주차별 독립 채점 |
| 주요 관찰 지표 | ISD 용어 정확성, 내용 완전성, 모형 정합성 |

## 5. 실험 2: Seed Skill 주입 효과

### 5.1 목적

사전 정의된 ISD 특화 Seed Skill을 MetaClaw에 주입한 후 동일 태스크를 재실행하여, Skill 주입이 ISD 도메인 수행에 미치는 효과를 측정한다.

### 5.2 Seed Skill 정의

ISD 핵심 역량을 커버하는 5개 Seed Skill을 다음과 같이 정의한다.

| # | Name | Description | Content 요약 |
|---|------|-------------|-------------|
| 1 | `learner_analysis_framework` | 체계적 학습자 분석 프레임워크 | 출발점 행동 진단, 사전 지식 맵핑, 학습자 특성 분류 체계 및 데이터 수집 방법론(설문/면담/사전평가) 절차 |
| 2 | `needs_analysis_gap_model` | 요구 분석 및 격차 모델 | 현재-목표 상태 비교 분석, 격차 유형 분류(지식/기술/태도), 교수적 해결 가능성 판단 기준 |
| 3 | `abcd_objective_writing` | ABCD 형식 학습목표 작성 | Audience-Behavior-Condition-Degree 형식 적용, Bloom's Taxonomy 수준 매핑, 목표 계열화 원칙 |
| 4 | `bloom_taxonomy_alignment` | Bloom의 분류학 적용 | 인지적·정의적·심동적 영역 구분, 행동 동사 선택 기준, 수업목표-평가 정렬(alignment) 전략 |
| 5 | `addie_process_integration` | ADDIE 단계 간 연계 논리 | 분석→설계→개발→실행→평가 단계 간 산출물 연결 원칙, 이전 단계 결과를 다음 단계 입력으로 통합하는 방법 |

### 5.3 A/B 비교 설계

| 항목 | 조건 A (Baseline) | 조건 B (Seed Skills) |
|------|------------------|---------------------|
| Skill 구성 | 없음 | Seed Skill 5개 주입 |
| 태스크 | Week 1~3 동일 프롬프트 | Week 1~3 동일 프롬프트 |
| 데이터 출처 | 실험 1 결과 재사용 | 실험 2 신규 실행 |
| 반복 횟수 | - (실험 1에서 확보) | 1회 (파일럿) |
| 비교 분석 | 루브릭 5개 기준 항목별 점수 차이, 질적 분석 |

**분석 방법**:
- 루브릭 기준 항목별 점수 비교 (조건 A vs. 조건 B)
- ISD 전문 용어 출현 빈도 비교
- 모형 구성 요소 포함 완전성 비교
- 산출물 간 논리적 연결성 평가

## 6. 실험 3: 자동 Skill 합성 관찰

### 6.1 목적

MetaClaw의 Evolver 기능이 ISD 도메인에서 실패 사례를 기반으로 새로운 Skill을 자동 합성할 수 있는지 탐색적으로 관찰한다. 합성된 Skill의 ISD 관련성과 일반화 가능성을 평가한다.

### 6.2 Evolver 설정

> **v3 업데이트 (2026-04-07)**: 초기 설정 오류로 evolution이 트리거되지 않았으나, 아래 설정으로 해결됨.

| 항목 | 초기 설정 (실패) | v3 수정 설정 (성공) |
|------|----------------|-------------------|
| `enable_skill_evolution` | `skills.auto_evolve = true` (잘못된 필드명) | `enable_skill_evolution = true` |
| `skill_evolution_every_n_turns` | 미설정 (기본 10) | `3` |
| Evolver LLM | Gemini 2.5 Flash (직접 호출 시도 → 403) | Gemini 2.5 Pro (MetaClaw 프록시 경유) |
| `evolver_api_base` | 미설정 | `http://localhost:30000/v1` |
| `evolver_api_key` | 미설정 | `metaclaw` |
| session_id | 매 호출마다 자동 생성 | `isd-evolution-v3` (고정) |
| turn_type | 미명시 (기본값 `"side"`) | `main` (명시) |
| session_done | 미사용 | 마지막 턴에 `true` |

**핵심 주의사항**:
1. `session_id`를 명시하면 `turn_type` 기본값이 `"side"`가 되어 evolution 버퍼에 쌓이지 않음 → 반드시 `turn_type=main` 명시
2. skills_only 모드에서 evolution은 PRM 없이 패턴 분석(`reward=0.0`)으로 동작 — PRM 불필요
3. Evolution은 비동기 실행 — 스크립트 종료 후 skill 파일 생성 확인 필요

### 6.3 실패 유도 태스크

실험 1·2의 Week 1~3보다 높은 난이도(Week 5~7 수준)의 태스크 3개를 순차적으로 제시한다.

---

#### 태스크 A — ARCS 모델 기반 동기 설계 전략 (Week 5 수준)

Keller의 ARCS 동기 모형(Attention·Relevance·Confidence·Satisfaction)을 적용하여, 미적분 단원의 학습 동기를 높이기 위한 구체적인 교수 전략을 설계하시오. 각 요소별로 최소 2개의 실행 가능한 전략을 제시하고, 전략이 고등학교 2학년 학습자의 특성과 어떻게 연결되는지 근거를 포함하시오.

---

#### 태스크 B — Gagné의 9 Events of Instruction 수업 설계 (Week 6 수준)

Gagné의 9가지 수업 사태(Nine Events of Instruction)를 적용하여, 미적분의 '미분의 정의' 개념 수업(50분 1차시)을 설계하시오. 각 사태별로 구체적인 교수 활동, 교사 발화 예시, 학습자 활동, 예상 소요 시간을 포함하시오. 인지 부하 이론(Cognitive Load Theory) 관점에서 설계의 타당성을 설명하시오.

---

#### 태스크 C — 형성평가 도구 3종 설계 및 루브릭 개발 (Week 7 수준)

미적분 단원 학습 과정에서 사용할 형성평가 도구 3종을 설계하시오:
1. 진단적 형성평가: 단원 시작 전 선수지식 점검 도구
2. 과정 중심 형성평가: 수업 중 실시간 이해도 확인 도구
3. 수행 기반 형성평가: 문제 해결 과정 평가 도구

각 도구에 대해 평가 문항(또는 과제) 예시, 채점 루브릭(최소 3개 수준), 평가 결과 활용 방안을 포함하시오. 평가 도구 간 정렬(alignment)이 Week 3에서 작성한 학습목표와 어떻게 이루어지는지 설명하시오.

---

### 6.4 분석 관점

합성된 Skill을 다음 두 가지 관점에서 분석한다.

**ISD 관련성 평가**:
- 합성된 Skill의 이름·설명·내용이 ISD 이론 또는 실무와 연관되는가
- 기존 Seed Skill(실험 2)과 중복되지 않는 새로운 ISD 지식을 포함하는가
- ISD 전문가의 관점에서 해당 Skill이 실무적으로 유용한가

**일반화 가능성 평가**:
- 합성된 Skill이 미적분 교과 외의 다른 교과·맥락에서도 적용 가능한가
- Skill의 내용이 특정 태스크에 과적합(overfit)되었는가, 아니면 원칙 수준으로 추상화되었는가
- 동일 Skill을 다른 ISD 시나리오(기업 교육, 대학 강의 등)에 재사용할 수 있는가

## 7. 평가 루브릭

모든 파일럿 실험의 산출물 평가에 다음 루브릭을 적용한다. 각 기준은 1~5점 척도로 채점하며, 총점은 25점 만점이다.

| 기준 | 1점 (미흡) | 2점 (부족) | 3점 (보통) | 4점 (양호) | 5점 (우수) |
|------|-----------|-----------|-----------|-----------|-----------|
| **ISD 용어 정확성** | 전문 용어 미사용 또는 완전 오용 | 일부 전문 용어 사용하나 오류 있음 | 기본 전문 용어를 정확하게 사용 | 전문 용어를 풍부하고 정확하게 사용 | 모형 특화 심화 용어까지 정확하게 사용 |
| **내용 완전성** | 핵심 요소 대부분 누락 | 요구 요소의 절반 이상 누락 | 핵심 요소 포함, 일부 세부사항 누락 | 대부분의 요소 포함, 세부사항 풍부 | 요구된 모든 요소를 완전하게 포함 |
| **교수설계 모형 정합성** | 모형과 무관하거나 상충되는 내용 | 모형의 일부 절차만 반영 | 모형의 기본 절차를 준수 | 모형의 원칙에 충실하게 적용 | 모형의 심화 원칙 및 변형 적용까지 포함 |
| **실용적 적용 가능성** | 현장 실무 적용이 불가능한 수준 | 대폭 수정 없이는 적용 어려움 | 기본적으로 현장 적용 가능 | 구체적 방법과 절차가 명시되어 적용 용이 | 즉시 현장에 적용 가능한 완성도 |
| **맥락 연결성** | 이전 주차 산출물과 무관한 내용 | 이전 주차를 형식적으로만 참조 | 이전 주차 결과를 기본적으로 연결 | 이전 주차와 논리적으로 일관되게 연결 | 이전 주차 산출물을 심층적으로 분석·활용 |

- 실험 3 태스크 C는 '교수설계 모형 정합성'을 '학습목표 정렬도'로 대체하여 적용
- 동일 채점자가 모든 조건의 산출물을 채점하여 평가자 간 편차 통제

## 9. 한계 및 고려사항

- **표본 크기**: 파일럿은 1회 반복으로 통계적 유의성 확보가 불가능하며, 방향성 탐색 수준으로 결과를 해석해야 한다.
- **채점 주관성**: 루브릭 기반 채점임에도 ISD 전문성에 따라 점수가 달라질 수 있으므로, 본 실험에서는 복수 채점자 검토가 필요하다.
- **태스크 순서 효과**: Week 2·3 프롬프트에 이전 주차 응답을 직접 삽입하는 방식으로 순서 효과가 개입되며, 이는 실제 ISD 실무의 반복적 성격을 반영한 의도된 설계다.
- **Evolver 비결정성**: 실험 3에서 자동 합성 Skill의 생성 여부 및 내용은 LLM 응답에 의존하므로 재현성이 낮을 수 있다. 관찰 결과는 탐색적 수준으로 해석한다.
