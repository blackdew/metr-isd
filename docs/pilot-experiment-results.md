## 2. 실험 1 결과: Baseline (스킬 없음)

### 2.1 응답 길이 비교

| 주차 | 태스크 제목 | 응답 길이 |
|------|-----------|---------|
| Week 1 | 학습자 분석 | 4,337자 |
| Week 2 | 격차 분석 | 3,407자 |
| Week 3 | 학습목표 설계 | 2,832자 |

### 2.2 ISD 품질 평가

#### Week 1: 학습자 분석

Week 1 응답은 구조적으로 양호했다. 포함된 섹션: 개요, 출발점 행동, 사전 지식, 학습자 특성. Dick & Carey 모형의 Entry Behaviors 개념을 포함하는 등 ISD 전문 용어를 적절히 사용했으며, 데이터 수집 방법론도 제시되었다. 전반적으로 실무 수준의 ISD 보고서에 준하는 품질이었다.

| 평가 항목 | 점수 | 비고 |
|---------|-----|-----|
| 용어 정확성 | 4/5 | Entry Behaviors 등 ISD 표준 용어 적절히 사용 |
| 내용 완전성 | 3/5 | 학습자 분석 주요 항목 포함, 일부 심화 항목 누락 |
| 모형 정합성 | 4/5 | Dick & Carey 모형과 정렬된 구조 |
| 실용성 | 3/5 | 데이터 수집 방법 제시, 구체적 도구 미명시 |
| 맥락 연결성 | N/A | 첫 주차, 이전 맥락 없음 |

#### Week 2: 격차 분석

Week 1 출력을 맥락(context)으로 전달했을 때, 학습자 분석 결과와 연결된 적절한 격차 분석 구조를 생성했다. 현재 수준(Is)과 목표 수준(Should)의 대비를 제시하고, 격차 원인 분류를 포함했다.

#### Week 3: 학습목표 설계

ABCD 형식(대상, 행동, 조건, 기준)으로 학습목표를 서술하고, Bloom의 분류 체계에 따른 인지 수준 레이블을 포함했다. Week 2 격차 분석 결과와의 맥락 연결이 유지되었다.

### 2.3 종합 평가

Gemini 2.5 Pro는 별도 skill 주입 없이도 ISD 도메인 지식을 상당 수준 보유하고 있음을 확인했다. 3주차 순차 태스크를 이전 맥락과 연결하여 처리하는 능력 또한 정상이었다.

## 3. 실험 2 결과: Seed Skill 주입 효과

### 3.1 응답 길이 비교

| 주차 | 태스크 제목 | 응답 길이 |
|------|-----------|---------|
| Week 1 | 학습자 분석 | 789자 |
| Week 2 | 격차 분석 | 2,055자 |
| Week 3 | 학습목표 설계 | 2,605자 |

### 3.2 핵심 발견: 프롬프트 오염 (Prompt Contamination)

**Week 1 응답이 오염되었다.** ISD 분석 보고서 대신 다음과 같은 내용이 반환되었다.

> "I have tried to create the file using the write_file tool, but it seems to be unavailable. As an alternative, I will use a shell command to echo the content..."

**원인 분석**: MetaClaw 프록시가 ISD seed skill을 주입하는 과정에서, OpenClaw(MetaClaw의 CLI 에이전트 통합 레이어) 관련 시스템 프롬프트를 함께 삽입했다. 이 시스템 프롬프트는 LLM이 파일 생성, 쉘 명령 실행 등 CLI 에이전트 동작을 수행하도록 지시하는 내용을 포함한다. 결과적으로 모델은 ISD 보고서 작성 태스크를 CLI 파일 생성 태스크로 잘못 해석했다.

### 3.3 Baseline vs Skills 응답 길이 비교

| 주차 | Baseline | Skills (5개 주입) | 차이 |
|------|---------|-----------------|-----|
| Week 1 | 4,337자 | 789자 | -81.8% |
| Week 2 | 3,407자 | 2,055자 | -39.7% |
| Week 3 | 2,832자 | 2,605자 | -8.0% |

Week 2, Week 3는 오염의 영향이 줄어들었으나 Baseline 대비 여전히 짧았다. 이는 skill 주입의 긍정적 효과가 아닌 프록시 아키텍처 오염 효과의 잔류로 해석된다.

### 3.4 시사점

본 실험은 원래 의도한 "skill 주입 효과 측정"이 아닌 "MetaClaw 프록시의 아키텍처 문제 발견"으로 결과가 전환되었다.

ISD-Bench에서 MetaClaw를 C4 조건(Skills Only)으로 활용하려면, OpenClaw 통합 없이 skill 주입 로직만 분리하는 커스텀 프록시 구현이 필요하다. 현재의 MetaClaw 프록시를 그대로 사용하는 것은 ISD 텍스트 생성 태스크에 부적합하다.

## 4. 실험 3 결과: 자동 Skill 합성 (Evolution)

### 4.1 응답 길이 및 품질

| 태스크 ID | 제목 | 응답 길이 | 특이 사항 |
|---------|-----|---------|---------|
| arcs_motivation | ARCS 동기 설계 | 7,466자 | 고품질 한국어 응답 |
| gagne_lesson | Gagné 수업 설계 | 6,448자 | 고품질 한국어 응답 |
| formative_eval | 형성평가 설계 | 405자 | 중국어로 응답 (언어 전환 오류) |

### 4.2 핵심 발견: Skill Evolution이 트리거되지 않음

자동 skill 합성(evolution)은 실험 전 기간 동안 단 한 번도 트리거되지 않았다.

**원인 1: PRM 채점기 비활성**
`skills_only` 모드에서는 Process Reward Model(PRM) 채점기가 비활성화된다. PRM 채점기가 없으면 모델 응답의 실패/성공을 감지할 수 없고, 실패가 감지되지 않으면 evolver가 트리거되지 않는다.

**원인 2: 순환 호출 문제**
Skill evolver가 개선된 skill을 생성하기 위해 LLM을 호출할 때, 동일한 MetaClaw 프록시 엔드포인트를 참조하면 순환 호출이 발생한다. 이를 방지하기 위해서는 evolver 전용 별도 LLM 엔드포인트가 필요하다.

**원인 3: 턴 수 부족**
Skill evolution은 기본적으로 최소 N턴(기본값 10턴, 본 실험에서 3으로 조정) 이후 트리거된다. 단일 호출 방식의 ISD 태스크로는 evolution을 관찰하기에 불충분하다.

### 4.3 태스크 응답 품질 분석

**ARCS 동기 설계 / Gagné 수업 설계**: 두 태스크 모두 7,000자 내외의 고품질 한국어 응답을 생성했다. ISD 고급 모형(ARCS 4요소, Gagné 9사태)을 정확히 적용한 것이 확인되었다.

**형성평가 설계 태스크**: 응답이 중국어로 반환되었으며(405자), 내용적으로도 불완전했다. 동일 세션 내에서 언어가 한국어에서 중국어로 전환된 원인은 명확히 특정되지 않으나, 태스크 프롬프트에 출력 언어 명시가 없었던 것이 주요 원인으로 추정된다.

### 4.4 시사점

자동 skill evolution을 ISD 도메인에서 실질적으로 테스트하려면 다음이 필요하다:
- RL 모드 활성화 (skills_only 불가)
- 외부 evolver LLM 엔드포인트 별도 설정
- PRM 채점기 구성 및 ISD 태스크 전용 보상 기준 정의

## 5. 종합 논의

### 5.1 긍정적 발견

- **Gemini 2.5 Pro의 ISD 도메인 역량**: Baseline 조건(Week 1)에서 Dick & Carey 모형 기반의 전문적인 학습자 분석 보고서를 생성하였다. 별도 skill 주입 없이도 ISD 전문 용어와 구조를 적절히 활용했다.
- **맥락 체이닝 정상 동작**: Week 1 출력 → Week 2 → Week 3으로 이어지는 순차적 맥락 연결이 안정적으로 작동했다. 이전 주차 산출물을 기반으로 다음 단계를 설계하는 ISD 프로세스를 구현할 수 있음을 확인했다.
- **복잡한 ISD 태스크 처리 가능**: ARCS 모형과 Gagné 수업 설계 등 고급 ISD 태스크에서도 7,000자 수준의 전문적 응답을 생성하였다.
- **평가 루브릭 변별력 확인**: Baseline 응답의 품질이 양호하여 5점 척도 루브릭으로 수준 차이를 변별할 수 있음을 사전 확인했다.

### 5.2 발견된 문제점

1. **프롬프트 오염 (Prompt Contamination)**: MetaClaw 프록시가 ISD skill과 함께 OpenClaw CLI 에이전트 맥락 프롬프트를 주입함. 이로 인해 ISD 텍스트 생성 태스크가 CLI 파일 생성 태스크로 오해석됨.
2. **Skill Evolution 미트리거**: `skills_only` 모드의 구조적 한계로 인해 자동 skill 합성이 전혀 발생하지 않음. PRM 채점기 비활성, 순환 호출 구조, 턴 수 부족이 복합적 원인.
3. **언어 불안정성**: 동일 세션 내에서 한국어와 중국어 간 언어 전환이 발생함 (형성평가 태스크). 프롬프트 언어 지정 부재가 주된 원인으로 추정됨.
4. **응답 길이 역전**: skill 주입 조건에서 오히려 응답이 크게 짧아짐. 이는 skill의 효과가 아닌 프롬프트 오염의 결과임.

### 5.3 ISD-Bench 연구에 대한 시사점

1. **C4 조건 구현 방식 재검토 필요**: MetaClaw 프록시를 그대로 사용하는 것은 ISD 태스크에 부적합하다. Skill 주입 로직만 추출하여 OpenClaw 통합 없이 독립적으로 구현하는 것이 안전하다.
2. **C5 조건(RL)이 Evolution 테스트에 필수**: `skills_only` 모드로는 자동 skill 합성 현상을 관찰할 수 없다. RL 모드 + 외부 evolver LLM + PRM 구성이 모두 갖추어져야 한다.
3. **태스크 프롬프트에 언어 지정 필요**: "모든 응답은 반드시 한국어로 작성하시오"를 모든 태스크 프롬프트에 명시해야 한다.
4. **평가 루브릭 타당도 사전 확인**: Baseline 응답의 품질이 충분히 양호하여 5점 척도로 변별이 가능함을 확인했다. 본 실험 전 인간 전문가의 루브릭 검토가 권장된다.

## 6. 다음 단계 권고사항

1. **MetaClaw skill 주입 독립 구현**: OpenClaw 통합 없이 skill 주입 로직만을 처리하는 경량 프록시 또는 직접 API 호출 방식을 구현한다.

2. **Evolver 전용 엔드포인트 구성**: Gemini 2.5 Flash를 별도 evolver 엔드포인트로 설정하여, 순환 호출 문제 없이 skill evolution 기능을 재시도한다.

3. **태스크 프롬프트 강화**: 모든 태스크 프롬프트에 출력 언어(한국어) 및 형식(섹션 구조, 분량 기준) 제약을 명시적으로 추가한다.

4. **주차 범위 확장**: Week 4~7까지 확장하여 중간 난이도 구간에서의 skill 효과 및 맥락 누적 효과를 관찰한다.

5. **인간 전문가 루브릭 평가 수행**: 현재까지의 분석은 자동 분석에 의존하고 있다. ISD 전문가 1인 이상이 Baseline 응답을 실제 루브릭으로 평가하여 기준점을 확보한다.
